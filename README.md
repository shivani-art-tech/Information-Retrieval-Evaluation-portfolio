# Information Retrieval Evaluation Portfolio

## Overview

This repository demonstrates Information Retrieval (IR) Evaluation skills used in AI search systems, enterprise search engines, Retrieval-Augmented Generation (RAG) pipelines, research assistants, knowledge management systems, and Large Language Model (LLM) evaluation projects.

The portfolio showcases my ability to assess search result relevance, source quality, retrieval accuracy, evidence coverage, information completeness, and user intent satisfaction.

As a reviewer, I focus not only on whether information is technically correct, but also whether the retrieved information genuinely helps the user accomplish their goal.

# Sample 01: Relevant but Incomplete Retrieval

## User Query

What are the major causes of air pollution in Indian cities?

## Retrieved Result

A report discussing vehicle emissions as a contributor to urban pollution.

## Human Evaluation

### Assessment

The result is relevant because vehicle emissions are a major source of pollution.

However, the search result fails to address other important contributors such as industrial emissions, construction dust, biomass burning, and waste combustion.

A user seeking a comprehensive answer would not receive sufficient information.

### Relevance Score

7/10

### Coverage Score

4/10

### Verdict

Partially Relevant

### Reviewer Notes

The retrieval system identified a valid source but failed to provide adequate topic coverage.

# Sample 02: High Authority Source Selection

## User Query

What are the symptoms of dengue fever?

## Retrieved Results

Result A:
Personal blog discussing dengue symptoms.

Result B:
Official public health guidance describing symptoms, diagnosis, and treatment recommendations.

## Human Evaluation

### Assessment

Both results discuss dengue fever.

However, Result B originates from a reliable medical authority and provides evidence-based information.

For health-related queries, source authority significantly impacts retrieval quality.

### Preferred Result

Result B

### Reason

Higher reliability and greater user trust.

### Verdict

High Quality Retrieval

# Common Retrieval Errors

## Critical Errors

* Irrelevant retrieval
* Misinformation sources
* Unsupported claims
* Intent misunderstanding

## Major Errors

* Poor ranking
* Missing key information
* Outdated sources

## Moderate Errors

* Limited coverage
* Weak evidence supports

## Minor Errors

* Formatting issues
* Small ranking adjustments

# Quality Rating Scale

10 = Excellent

8–9 = Strong Retrieval

6–7 = Acceptable

4–5 = Needs Improvement

1–3 = Poor Retrieval

0 = Completely Irrelevant
